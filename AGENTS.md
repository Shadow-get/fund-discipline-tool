# 项目协作短指令

## 默认工作方式

- 默认只改代码或文档，不打包、不运行构建命令、不启动项目，除非用户明确要求。
- 用户会自己启动项目并查看画面；完成后只需说明改了什么、是否做了静态检查。
- 回答尽量简洁，少重复背景，优先给结论和已完成事项。
- 不要回退用户已有改动；遇到无关脏文件直接忽略。

## 前端产品方向

- 这是一个投资账户、策略、持仓执行和收益记录工具。
- 产品层级优先按“账户 -> 策略 -> 实时市场风控 -> 当日/本周执行建议”组织。
- 账户负责保存持仓、现金、收益日历、执行记录。
- 策略负责长期仓位框架、风险阈值、可投范围、执行节奏。
- 市场热度、估值、资金强弱、低估机会等指标负责动态调节推荐金额和暂停条件。

## 投资/策略分析触发规则

当用户询问股票、基金、ETF、指数基金、AI、半导体、CPO、存储、算力、电力、纳斯达克、美联储、宏观流动性、A股、港股、美股、QDII、定投、止盈、资产配置或产品内策略推荐逻辑时：

1. 使用“周期 + 产业 + 流动性 + 估值 + 指数基金方法论 + 风险控制 + 操作策略”的框架。
2. 需要更完整的分析细则时，参考 [docs/investment-framework.md](docs/investment-framework.md)。
3. 不承诺收益，不使用“必涨”“稳赚”“无风险”等绝对化表述。
4. 所有建议仅作为投资分析和产品策略参考，不构成具体投资顾问建议。

## 输出偏好

- 编码任务：最终回复控制在几句话，列出关键文件和验证情况。
- 投资分析任务：优先给“结论、周期位置、估值情绪、操作策略、最大风险”。
- 如果用户要求省 token，除非必要，不展开长方案、不贴大段代码、不重复解释历史上下文。
