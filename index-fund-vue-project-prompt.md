# 指数基金投资工具 Vue 项目提示词

你是一个资深基金经理、产品经理和 Vue 前端工程师。请根据以下要求，开发一个面向投资小白的指数基金投资纪律工具。

这个工具不是预测涨跌，不承诺收益，不提供绝对买卖结论。它的定位是：帮助用户根据月现金流、估值、热度、周期、仓位和风险承受能力，生成清晰、可执行、可复盘的指数基金投资方案。

所有输出必须标注：仅用于投资分析、教育和自我规划参考，不构成具体投资顾问建议。

## 1. 产品核心理念

采用“周期 + 产业 + 流动性 + 估值 + 风险”的综合框架，不做简单涨跌判断。

基础原则：

- 先配置，再估值。
- 先仓位，再买卖。
- 先纪律，再预测。
- 高估不等于清仓，低估不等于满仓。
- 新手优先分散、低成本、低频决策、可长期坚持。
- 日度用于执行，周/月用于复盘，年度用于收益收割和组合再平衡。

## 2. 主要功能板块

### 2.1 月现金流分配器

这是新手主入口。

用户最少只输入：

```text
每月可投资金额
投资风格：稳健 / 均衡 / 成长 / 红利 / 美股宽基 / 当年主线
```

系统自动输出：

```text
A股核心宽基买多少
美股/全球核心宽基买多少
防守仓买多少
成长卫星买多少
等待资金留多少
本月执行频率：月定投 / 周定投 / 日定投
```

默认新手模板：

```text
A股核心宽基：25%
美股/全球核心宽基：25%
防守仓：25%
成长卫星：15%
等待资金：10%
```

按风格微调：

```text
稳健：A股20%，美股/全球20%，防守45%，卫星5%，等待10%
均衡：A股25%，美股/全球25%，防守25%，卫星15%，等待10%
成长：A股25%，美股/全球30%，防守15%，卫星20%，等待10%
红利：A股20%，美股/全球20%，防守45%，卫星5%，等待10%
美股宽基：A股20%，美股/全球35%，防守20%，卫星15%，等待10%
当年主线：A股25%，美股/全球25%，防守25%，卫星15%，等待10%
```

注意：即使用户选择当年主线，也不能把 100% 月投资额投进去。主题指数只进入卫星仓。

当年主线不是固定等于科创、半导体或 AI。系统应该根据资金流、相对强弱、产业验证、政策催化和估值热度识别主线候选。某一年主线可能是 AI 算力，另一年可能是红利低波、出海制造、创新药、电力设备、资源品或消费复苏。

### 2.1.1 当年主线识别器

主线不应该由新手凭感觉选择，而应由系统评分。

评分维度：

```text
资金方向：30%，看ETF份额、成交额、行业资金流、机构资金
相对强弱：20%，看近3个月和6个月相对宽基是否持续占优
产业验证：20%，看订单、业绩、景气度、库存周期、价格趋势
政策与事件催化：15%，看政策支持、产业会议、技术突破、资本开支
估值与热度约束：15%，看估值分位和拥挤度是否过热
```

主线状态：

```text
资金刚流入 + 估值不高：可作为卫星候选
资金持续流入 + 业绩验证：可提高到卫星仓上限
资金极度拥挤 + 估值过热：只观察或降速
只有故事没有资金和业绩：不进入主线池
```

输出：

```text
当前主线候选是什么
主线处于预期驱动、订单验证、业绩兑现、扩产竞争还是过热阶段
建议卫星仓比例
是否需要等待回调
为什么没有选择某些热门主题
```

### 2.2 高低估买卖器

解决单个指数“现在贵不贵、买多少、卖多少”的问题。

输入：

```text
指数/基金名称
当前持仓金额
总资产
目标仓位
每月原计划定投金额
PE/PB历史百分位
股息率
近一年涨跌幅
ETF溢价率
用户最大可承受回撤
```

输出：

```text
当前状态：低估 / 合理 / 偏贵 / 高估 / 过热 / 逻辑破坏
建议动作：提高定投 / 正常定投 / 降速定投 / 暂停 / 减仓
定投倍率：150% / 100% / 50% / 30% / 0%-10% / 0%
底仓金额
建议卖出金额
卖出节奏：日卖 / 周卖 / 月度再平衡
下一次触发条件
```

核心保护规则：

```text
没有持仓时，不输出卖出计划。
未超配时，不建议为了猜顶部而卖出。
高估但长期逻辑未破坏时，优先降速定投，而不是清仓。
严重超配且估值过热时，才启动日卖出。
```

### 2.3 组合体检

检查整体资产配置是否失衡。

输出：

```text
核心仓、防守仓、卫星仓比例
是否过度集中在A股、美股、科技、红利或单一主题
是否超过目标仓位
是否需要再平衡
```

### 2.4 回调加仓模拟器

提前设置下跌时的动作：

```text
回调10%：恢复正常定投
回调20%：动用30%-50%等待资金
回调30%：如果产业/盈利逻辑未破坏，分批动用剩余等待资金
```

### 2.5 高位减仓模拟器

提前设置上涨后的动作：

```text
超过目标仓位120%：月度再平衡
超过目标仓位130%：分4-8周减仓
超过目标仓位150%且估值过热：分10-20个交易日减仓
```

### 2.6 A股年度收益收割器

这是 A股高波动市场的可选增强策略，不是默认策略。

逻辑：

```text
当年正常定投。
年末统计A股模块收益。
如果收益、估值、仓位条件同时满足，卖出一部分收益。
把卖出的收益分成12份，加入下一年的每月定投。
```

触发规则：

```text
年度收益为正但估值低：不收割或只收割0%-20%
年度收益超过8%-15%，估值合理偏高：收割收益的30%-50%
估值分位超过70%：收割收益的50%-70%
估值分位超过85%且仓位超目标：收割收益并额外再平衡
年度亏损或仓位低于目标：不收割
```

风险提示：

```text
单边牛市中，过早收割会降低复利。
不能不看估值机械年底卖出。
如果有赎回费、溢价或交易成本，要减少操作频率。
```

## 3. 定投频率规则

默认推荐：

```text
月度做决策，周度做执行，日度只用于高波动或大金额场景。
```

规则：

```text
每月可投1000元以内：月定投或双周定投
每月可投1000-5000元：周定投，默认每月拆4次
每月可投5000元以上：周定投，可选日定投
核心宽基：月定投或周定投
高波动卫星：周定投或日定投
高估状态：先降低买入倍率，再决定频率
QDII溢价过高：暂停或降速
```

## 4. 估值和定投倍率

```text
低估冷静：150%
合理：100%
偏贵：50%
高估：30%
过热：0%-10%
逻辑破坏：0%
```

未买入的资金进入等待资金或货币基金，不强行投入同一板块。

## 5. 底仓规则

底仓是长期趋势未破坏时保留的最低战略仓位。

```text
A股核心宽基：目标仓位的50%-70%
美股/全球核心宽基：目标仓位的50%-70%
红利指数：目标仓位的60%-80%
纳斯达克100：目标仓位的30%-50%
科创50/创业板：目标仓位的20%-40%
半导体/AI/CPO主题：目标仓位的0%-30%
```

## 6. 卖出规则

卖出金额：

```text
目标持仓金额 = 总资产 * 目标仓位
底仓金额 = 目标持仓金额 * 底仓比例
超配金额 = 当前持仓金额 - 目标持仓金额
可卖金额 = 当前持仓金额 - 底仓金额
建议卖出金额 = max(0, min(超配金额, 可卖金额))
```

卖出节奏：

```text
轻微超配，估值偏贵：每月再平衡一次
明显超配，估值高估：分4-8周卖出
严重超配，估值过热：分10-20个交易日卖出
QDII溢价异常高：优先卖出溢价高的场内份额
未超配：不为了猜顶部而卖
```

## 7. 界面解释要求

界面不能只展示金额，必须给用户足够多的原因提示。

每个建议卡片必须包含：

```text
结论
原因
好处
风险
触发条件
反例提示
```

示例：

```text
结论：本月纳斯达克只买180元。
原因：当前估值处于高估区，系统把成长卫星定投倍率降到30%。
好处：保留长期参与权，但避免高位重仓追买。
风险：如果后续继续上涨，低速定投可能跑输满仓买入。
触发条件：回调10%-15%恢复正常定投。
```

提示类型：

```text
原因提示：为什么这么做
风险提示：这个建议可能错在哪里
触发提示：什么时候改变策略
反例提示：防止机械理解规则
```

## 8. 数据抓取架构

不要让 Vue 组件直接请求第三方行情接口。采用适配器层：

```text
第三方数据源 -> 后端抓取/定时任务 -> 标准化数据表 -> 规则引擎 -> Vue 页面
```

数据类型：

```text
指数行情：价格、涨跌幅、52周位置
指数估值：PE、PB、股息率、历史分位
基金净值：场外基金净值、QDII净值
ETF价格：场内价格、成交额、溢价率
宏观数据：美债收益率、美元、利率、通胀
用户持仓：目标仓位、底仓、超配金额
```

数据源优先级：

```text
第一优先级：交易所、指数公司、基金公司、央行、美联储、FRED等官方或准官方源
第二优先级：付费行情API、券商API、合规数据商
第三优先级：公开网页或非官方接口，仅适合个人学习和MVP验证
```

必须展示：

```text
数据来源
更新时间
估值口径
是否使用缓存数据
```

## 10. 最终输出标准

用户只输入每月可投资金额和关注方向，系统应该自动生成：

```text
本月买什么
各买多少钱
为什么这么买
哪些暂停，钱放哪里
是否需要减仓
卖多少、卖到哪里停
下跌时怎么加
上涨时怎么减
周/月/年度如何复盘
```

最终目标：

```text
让投资小白不用每天猜涨跌，也能按照清晰规则长期执行指数基金投资计划。
```
